Projet 1 : Étude de Cas Complète - Intrusion en Environnement Contrôlé

Contexte et Objectif : Ce projet illustre l'intégralité d'un cycle de test d'intrusion, depuis la construction d'un laboratoire sécurisé jusqu'à l'exploitation d'une vulnérabilité ciblée. L'objectif est de démontrer une maîtrise méthodologique des outils et des concepts fondamentaux du hacking éthique.

Partie 1 : Construction du Laboratoire de Pentesting

La première étape indispensable a été de créer un environnement de travail isolé et sécurisé pour mener mes expériences sans aucun risque pour mon réseau local ou pour Internet.

  • Virtualisation : J'ai utilisé Oracle VirtualBox pour déployer deux machines virtuelles distinctes.
  • Machine Attaquante : Une installation standard de Kali Linux, la distribution de référence pour les tests d'intrusion.
  • Machine Cible : Une machine Metasploitable2, conçue spécifiquement pour l'apprentissage avec ses vulnérabilités connues.
  • Isolation Réseau : Les deux machines ont été configurées sur un "Réseau Interne" (intnet) de VirtualBox. Cette configuration crée un réseau local virtuel totalement coupé du monde extérieur, garantissant une sécurité maximale.

Partie 2 : Déroulement du Test d'Intrusion (Write-up)

Une fois le laboratoire opérationnel, j'ai procédé à l'attaque en suivant une méthodologie structurée.

  1. Reconnaissance & Énumération :

    Un scan Nmap approfondi (nmap -sV -p- 192.168.56.11) a révélé un service distcc v1 sur le port 3632, une porte d'entrée potentielle connue pour sa vulnérabilité d'exécution de code à distance (RCE).

  2. Exploitation avec Metasploit et Résolution de Problème :

    J'ai utilisé le module exploit/unix/misc/distcc_exec de Metasploit. Après un premier échec dû à un payload incompatible, j'ai analysé les options disponibles et sélectionné cmd/unix/reverse_perl, un payload plus robuste qui établit une connexion inversée de la victime vers ma machine Kali.

  3. Résultat & Accès Initial :

    Après configuration des adresses RHOSTS et LHOST, l'exploitation a réussi. J'ai obtenu un shell sur la machine cible avec les privilèges de l'utilisateur daemon, marquant ainsi le succès de la phase d'intrusion et la première étape avant une potentielle escalade de privilèges.

Compétences Transversales Démontrées :
Virtualisation (VirtualBox) Configuration Réseau Nmap (Scanning) Metasploit Framework Analyse de Vulnérabilités Résolution de Problèmes

Projet 2 : Analyse de Logs de Sécurité

Objectif : Appliquer mes compétences en analyse de données (issues de mon Master MIASHS) à une problématique de cybersécurité.

Description : En utilisant un jeu de données public de logs de serveur web, j'ai développé des scripts pour parser et analyser les informations. Le but était d'identifier des anomalies telles que des tentatives de connexion par force brute, des balayages de ports ou des requêtes suspectes (injections SQL). Ce projet m'a permis de comprendre l'importance de l'analyse de logs dans la détection d'incidents.

Outils utilisés : Excel pour une première analyse, Python (librairie Pandas) pour le traitement avancé.
